# Job Pipeline — User Configuration

> This is a **template**. The onboarding wizard (`npm run dev`) generates your own `USER_CONFIG.md` with real values when you walk through the steps.
> Use this file only if you prefer to hand-edit and skip the wizard.
> Copy it to `USER_CONFIG.md` at the repo root, then fill in your values.

## Google IDs

| Setting | Value |
|---------|-------|
| Resume Google Doc ID | <resume-doc-id> |
| Job Tracker Google Sheets ID | <tracker-sheet-id> |
| Resume Drive Folder ID | <drive-folder-id> |

How to get each ID: open the Doc / Sheet / Folder in Google Drive and copy the long string from the URL (e.g. `https://docs.google.com/document/d/<this-part>/edit`).

### Column mapping

| Field | Column |
|-------|--------|
| Company | J |
| Role | K |
| Compensation | L |
| Date Posted | M |
| Job Post URL | N |
| Tier | O |
| Reason / Score summary | P |
| Full breakdown | Q |
| InMail | R |
| Connection Note | S |

These defaults match the shared template Sheet. If you duplicate the template, leave them as-is. If you use your own Sheet, update the letters to match your columns.

| Purpose | Column |
|---------|--------|
| Job URL (primary lookup) | N |

## Resume Section Anchors

Used by `customize-resume` to locate sections in your Google Doc. Each value is the first few unique words of the section in your resume — the wizard auto-detects these; override only if needed.

| Section | Opening string |
|---------|----------------|
| Summary paragraph | <first 6-8 words of your summary paragraph> |
| PROFESSIONAL EXPERIENCE section | PROFESSIONAL EXPERIENCE |
| KEY SKILLS line | <first 5 words of your skills line> |
| KEY SKILLS bold label | <e.g. "Design & Research:" or "" if none> |

---

## Resume File Naming

| Setting | Value |
|---------|-------|
| Filename pattern | <Your Name>_Resume_[Company]_[Role] |

The pipeline replaces `[Company]` and `[Role]` per job when saving tailored resumes to Drive.

## Pipeline Telemetry (Opt-In)

Anonymous usage data (step durations, token counts, job tier distribution) helps improve this tool.
No PII, resume content, job titles, or company names are ever sent.
Set "Share anonymous metrics" to `false` to disable at any time.

| Setting | Value |
|---------|-------|
| Share anonymous pipeline metrics | true |
| Anonymous user ID | <generated by wizard, e.g. usr_xxxxxxxxxxx> |
| PostHog key | <phc_… key, or "none" to disable> |
| Skill files hash | none |
